\subsection{Detailed Description}
\mbox{\hyperlink{classConfigFile}{Config\+File}} reads and writes data from a json configuration file. 

\mbox{\hyperlink{classConfigFile}{Config\+File}} provides an abstract basis for \mbox{\hyperlink{classResourceManager}{Resource\+Manager}} shared\+Resources that access .json file data. Each \mbox{\hyperlink{classConfigFile}{Config\+File}} should provide access to a set of key Strings for accessing its specific data. A default version of each \mbox{\hyperlink{classConfigFile}{Config\+File}}\textquotesingle{}s .json resource should be placed in the configuration subdirectory of the asset folder.

Along with reading and writing data, \mbox{\hyperlink{classConfigFile}{Config\+File}} objects allow Configurable objects to register to receive notification whenever particular data keys are changed.

\mbox{\hyperlink{classConfigFile}{Config\+File}} reads from each json file only once per program instance, so any external changes to the file that occur after that will be ignored and overwritten. To reduce file IO and prevent concurrent file modification, only one \mbox{\hyperlink{classConfigFile}{Config\+File}} object should exist for each json configuration file.
